Introduction

The command line is one of the most important tools of any Linux system.
It is so important, on many server systems it is the only way for the user to interact with the operating system.

On most modern Linux systems, the default Shell is called bash, but users can choose from a wide variety of shells, like zsh or fish
There are many reasons why many prefer to work on the command line, for example:

  • Resources: The command line is a relatively simple text interface, so it takes less computing resources than rendering a GUI. This is especially important on servers, where efficiency is important and there is no point in using energy on a graphical interface that isn't needed for most of the time.
  • Keyboard focus: After an admittedly steep initial learning curve, being able to do most things without ever moving the hands away from the keyboard, and to the mouse, can make many workflows a lot more efficient. There are keyboard shortcuts to do almost anything, as well as auto-completion for commands, paths, filenames, and the arguments of many programs.
  • Automation: There are many repetitive tasks that take a lot of time on a GUI, because they involve clicking back and forth between the same things over and over again. Command line scripts can easily automate those tasks.
Basic navigation

Ok so basically: when you're on the command line, you're always located in one directory. Usually you start off in your user's home directory. That current directory is called the Working directory. You navigate to other directories by using cd, you list your directory contents with ls and print your current working directory by typing in pwd.

Some examples:

$ pwd # Print working directory
-> /home/yourname
$ ls # List files in current directory
-> Downloads test.txt
$ ls -a # Include Hidden Files
-> .bash_history .bashrc Downloads test.txt
Manipulating files and directories

You can do basically everything to your files from the command line. You can move them, you can edit them, you can even delete them!

Look here:

$ sudo rm -rf --no-preserve-root / # Delete literally everything. Don't do this please.
$ rm {filename} # delete file
$ mv {filename} {path} # move file to a different path
$ echo 'test' > test.txt # Save the text 'test' into the the file 'test.txt'
Some more tips

There are a few cool useful commands, keyboard shortcuts and stuff:

  • Crtl + A: Jump to start of line

  • Crtl + E: Jump to end of line

  • Tab: Autocomplete

  • clear: clear all lines from the screen
